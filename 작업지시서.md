## 1. 불변 원칙 (반드시 유지)

1. **데이터를 열어 확인하고, 실행해서 검증한다.**
2. **엔진이 유일한 계산 출처.** 엑셀·앱·리포트가 다른 값을 내면 엔진이 맞다. 병렬 계산기(엑셀 수식, JS 계산) 금지 — 새 기능은 항상 "새 엔진 모듈 + 자체 테스트 + 근거"로만 추가한다.
3. **근거 없는 값 금지.** 모든 단가·규격·상수는 근거 명시, 확보 전까지는 `[추정]`/`[확인요망]` 표기.
4. **실측 벤치마크는 회귀 기준.** 밴드 안에 들어와야 정상.
5. **시세성 값(단가·노임·유가)은 엔진이 조회하지 않고 인자로 주입한다.**

쪼개고 자동화하는 기준은 "업무단계"가 아니라 **신뢰 유형**이다:

| 신뢰 유형 | 정의 | 처리 |
|---|---|---|
| 결정론(Deterministic) | 입력이 같으면 항상 같은 값 | 엔진이 계산 (자동화 O) |
| 시세성(Volatile) | 단가·노임·유가처럼 시점마다 변함 | 주입만 (자동화 X, 인터페이스만) |
| 판단성(Judgment) | 내재해 지정값 해석·작목 선정·부지 최종선정 | 보조/초안까지만 (자동화 X) |

이 경계선이 자동화 가능선이다. 넘어서 자동화하면 "숫자를 믿을 수 없고 추적할 수 없다"는 근본문제가 재발한다.

## 7. 미해결 이슈·리스크 (중복 제거 통합)

- ✅ **`U_VALUE` 전체 재조사 완료(2026-07-19, 라운드1~8), 유리=5.3 유지·필름 계열은 5.7→2.66 교체**: 유리(5.3)는 국토부 별표3 "단창" 표와 소수점까지 일치해 출처 오염 개연성은 확정적이나, 독립 물성검증(4.3~5.5kcal 범위)에서 물리적으로 타당해 값 유지("미검증(출처오염 개연성 높음)"). 필름·불소필름·단동(종전 5.7)은 PO/PE 재질 구분 없이 하나로 묶은 게 물리적으로 타당함(표면공기막 저항 지배)을 재확인했으나, 우민재 케이스 원문(공사설명서 p.1 "4)피복공사")을 직접 열람해 실제로는 단일층이 아닌 이중층(외피0.15T+내피0.1T) 피복임을 확정. 이중층 U값 국내1차출처(KISTI TRKO202100009930 「온실 열손실 저감 및 차단 기술 연구」)는 존재를 확인했으나 KISTI/NTIS 로그인장벽으로 끝내 확보 실패(NTIS 로그인은 성공했으나 NTIS 자체엔 원문 PDF가 없고 ScienceON 별도 로그인 필요 — SSO 연동 안 됨). 대신 사용자가 직접 찾은 별개 국내 1차 출처(이현우 등, 시설원예·식물공장 22(3):270-278, 2013, koreascience.kr 무료 원문 — KISTI가 아닌 별도 오픈액세스 플랫폼)로 실측치 확보: 경기도 화성시 실험온실 겨울 2개월 실측 관류열전달계수 — 일중피복 3.09W/㎡K(≈2.66kcal), 관행 이중피복 2.12W/㎡K(≈1.82kcal), 모형계산과도 교차검증됨. 이 실측 일중피복값이 종전 이론계산 기반 5.7과 2배 이상 괴리되는 걸 발견 — "실측 우선" 원칙에 따라 사용자 승인 하에 **U_VALUE["필름"/"불소필름"/"단동"]을 5.7→2.66으로 전면 교체**, **U_VALUE["필름_이중"]=1.82(신규 키)**를 실측치로 확정 반영(국제자료 기반 추정 3.8→실측 1.82). ⚠️ **알려진 트레이드오프(미해결 후속과제)**: `heating_load()`는 "최대난방부하"(장비용량 설계용 극한조건 기준) 계산인데 이현우 등 값은 "2개월 평균" 연료소비 기반 — 성격이 다른 두 개념에 같은 U를 쓰는 셈이라, 이 교체가 향후 필름 케이스의 난방장비 용량 추천을 과소산정할 위험이 있음(`heating_load()`가 safety=1.1을 곱하는 구조라 어느 정도 완충은 되나 확정 아님). ✅ **구조 분리 완료(2026-07-20)**: `heating_load()`가 `u_design`(최대부하용)·`u_period`(기간/연료소비용) 두 인자를 받도록 리팩터링했다. 미지정 시 둘 다 `U_VALUE[cover]`로 폴백해 기존 회귀값(원채원 케이스 등, `pytest` 79 passed로 재확인)은 완전히 그대로다 — 이번 변경은 값 교체가 아니라 "나중에 두 값을 따로 넣을 수 있는 구조"만 만든 것. `test_heating_load_u_default_stays_coupled`·`test_heating_load_u_design_period_separation` 회귀테스트 2건 추가. **여전히 미해결**: 극한설계조건용 `u_design`에 넣을 확정값(1차 출처)이 없어, 호출부가 명시적으로 지정하지 않는 한 안전마진 감소 위험은 구조적으로만 해소되고 실질적으로는 남아있다 — 원문서(엔진데이터 A-4/A-5 원본, 또는 KISTI TRKO202100009930)를 확보하면 그 값을 `u_design`에 주입할 것. 영향범위는 `cases/uminjae.json`(cover="필름") 1건뿐(원채원·춘천은 유리, 불소필름·이중층 쓰는 케이스 파일은 아직 없음). 상세는 `엔진데이터_레지스트리.json` U_VALUE source(라운드1~8 전체 이력) 참고.
